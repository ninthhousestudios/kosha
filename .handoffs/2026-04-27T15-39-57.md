## local infrastructure set up

### PyTorch path (works now)

The Qwen3-VL-Embedding repo is at `/home/josh/soft/Qwen3-VL-Embedding/` with:
- Model weights downloaded at `./models/Qwen3-VL-Embedding-2B/` (~4GB)
- Python venv managed by uv at `.venv/` (Python 3.13 via uv, not system 3.14)
- **CPU-only torch** installed — the default `uv sync` pulls CUDA torch which crashes on AMD. Fix was:
  ```
  uv pip install torch==2.8.0+cpu torchvision==0.23.0+cpu \
    --index-url https://download.pytorch.org/whl/cpu \
    --index-strategy unsafe-best-match
  ```
  The `--index-strategy` is needed because the repo's pyproject.toml has an aliyun mirror as first index.
- bf16 inference works: text 1.4s, image 253s, peak 4.6GB RAM

### fastembed/candle path (works with caveats)

Test project at `/home/josh/soft/fastembed-qwen3-test/` using a local patched fork at `/home/josh/soft/fastembed-local/`.

Three dtype fixes in `fastembed-local/src/models/qwen3.rs`:
1. Attention scale: cast to match attn dtype (line 818)
2. Attention mask: cast to match attn dtype (line 822)
3. L2 normalize eps: cast to match input dtype (line 151)

Filed upstream: https://github.com/Anush008/fastembed-rs/pull/248

**Caveat:** Only F16 works (candle lacks BF16 CPU matmul). F16 is faster than PyTorch (text 1.26s, image 139s, 3.87GB peak) but embedding quality degrades noticeably — relevant/irrelevant similarity ratio drops from 8x (BF16) to 2x (F16). BF16 is needed for production.
